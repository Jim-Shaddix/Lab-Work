0003 TDMS files can store many different types of data inside of them, such as 
0004 uint8, int16, doubles, etc. A full list is outside the scope of this document.
0005 
0006 One of these data types is rawDAQmx. I think this data type is actually the
0007 bit representation captured by the DAQ (hence the name raw). The presence
0008 of this data type in the meta for a TDMS file refers to the fact that the
0009 data has a true IEEE like data type underneath, as well as properties that
0010 are stored with the file that inform the user how to scale the data from
0011 the the bit-level representation to a true number.
0012 
0013 The primary reason one might use this format is because it gets rid of the
0014 intermediate step of converting the data to a known data type before
0015 saving. This could be advantageous because it could save computational
0016 time and memory from not doing the conversion, as well as time and disk space
0017 from writing a smaller file to disk.
0018 
0019 This data type however is not an open specification. In the TDMS file the
0020 header before the data will specify that upcoming data has this data type,
0021 allowing one to skip over that data section.
0022

0029 have a look through and hopefully it will answer some of your questions.
