0015 %5) TDMS does not currently support 2 dimensional arrays.  To save a 2d

0019 %6) National Instruments does not specify how chunks of data relate to

0023 %7) You can change the property value in a TDMS file, once it has
0024 %been written, but only the last property value is considered valid.  In
0025 %other words, let's say with every write, you save a boolean as to whether that
0026 %data is good or bad, in an associated write.  Thus, for every data write,
0027 %you have a property write.  First, there is no guarantee that each
0028 %changing of the property value will actually show up.  Second, there is no
0029 %guarantee that you will have the same # of property value changes as you
0030 %will # of data sections, to later match things up.  Thus, although it
0031 %might be possible to retain the property at different points in the file,
0032 %it most likely will not make sense, and thus only the last value is
